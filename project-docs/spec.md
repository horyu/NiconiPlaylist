# NiconiPlaylist 仕様書

本書はブラウザ拡張「NiconiPlaylist」の目的、ユーザーに見える挙動、外部ツールとの連携方針を整理する。

## 1. 概要

NiconiPlaylist は、ニコニコ動画に URL ベースの独自プレイリスト機能と連続再生機能を提供するブラウザ拡張機能である。

ユーザーまたは外部ツールが生成したプレイリスト共有 URL をインポートし、プレイリストを保存して連続再生できるようにする。

## 2. 背景と課題

ニコニコ動画公式には、マイリスト、検索結果、投稿者ページなど、特定スコープ内の動画を連続再生する機能が存在する。

しかし、公式機能で指定できる再生順は、公式が用意したソート順、またはランダムシャッフルに限定される。

そのため、以下のような任意の再生順を実現しにくい。

- 評価が高い動画と評価が不安定な動画を交互に並べる
- 外部で集計した評価指標や評価の安定性をもとに視聴順を設計する
- 特定の探索戦略に基づいて動画を並べる

また、そのような外部で生成したプレイリストを URL として共有する仕組みも公式機能だけでは扱いにくい。

さらに、公式のシャッフル再生はその場限りの順序に依存しやすく、数百件規模の動画を最後まで再生したり、途中から同じ順序で再開したりする用途に向かない。

NiconiPlaylist は、この制約を補い、URL ベースの任意順プレイリスト再生を提供する。

## 3. 目的

- URL によって動画プレイリストを共有できる
- URL に含まれる順番どおりにニコニコ動画を連続再生できる
- マイリストや検索結果など、公式が用意した動画一覧に依存しない任意順再生を提供する

## 4. 非目標

- NiconiCompare の評価計算機能を NiconiPlaylist 内に実装しない
- 動画メタデータの高度な分析やランキング生成は行わない
- クラウド同期やサーバー保存は行わない
- ニコニコ動画以外の動画サイトには対応しない
- 公式マイリストの編集や投稿者ページの並び替えを直接変更しない

## 5. 想定ユースケース

### 5.1 手動共有

ユーザーが動画 ID の列を含む URL を共有し、受け取ったユーザーがその URL を開くことで同じ順番で動画を視聴する。

### 5.2 外部ツール連携

外部ツールが生成したプレイリスト共有 URL をインポートして視聴する。

NiconiCompare 連携では、NiconiCompare の出力をもとに生成されたプレイリスト共有 URL をインポートし、評価指標に基づく順番で動画を視聴する。

## 6. プレイリスト URL 仕様

### 6.1 基本方針

プレイリスト共有 URL は、保存済みプレイリストをインポートするために使用する。

プレイリスト共有 URL は、query にエンコード済み ID 列と付加情報を載せる。

`videoIds` には、ID 列エンコード仕様に従った encoded string を指定し、`title` や `memo` などの付加情報は optional query として含める。

可読性の高い付加情報は前方に置き、長くなりやすい `videoIds` は query の末尾に置く。

`memo` は、タイトルだけでは表しきれないプレイリストの説明を共有するために用いる。

`title` や `memo` が共有 URL に含まれている場合は、インポート時にプレイリストへ同時に取り込む。

詳細は [ID 列エンコード仕様](./id-codec.md) を正とする。

### 6.2 URL 例

```txt
# タイトルなし動画ID 列エンコードのみ
https://horyu.github.io/NiconiPlaylist/?videoIds=<encoded>
# タイトルあり動画ID 列エンコード
https://horyu.github.io/NiconiPlaylist/?title=<title>&videoIds=<encoded>
# タイトルとメモあり動画ID 列エンコード
https://horyu.github.io/NiconiPlaylist/?title=<title>&memo=<memo>&videoIds=<encoded>
```

### 6.3 プレイリスト長制限

URL に載せられる長さには、ブラウザや共有経路ごとの制約がある。

そのため、最大 URL 長と最大動画件数は ID 列エンコード仕様ではなく、NiconiPlaylist の利用側仕様で制御する。

NiconiPlaylist は、生成または受信したプレイリスト共有 URL が利用側の上限を超える場合、プレイリストを無効としてユーザーに通知する。

### 6.4 GitHub Pages での共有URLプレビュー

GitHub Pages 上の `https://horyu.github.io/NiconiPlaylist/` は、共有 URL のプレビューと案内を兼ねる静的ページとして扱う。

- `videoIds` が無い場合は、拡張機能の案内とサンプル共有 URL を表示する
- `videoIds` がある場合は、共有 URL に含まれる `title` / `memo` / `videoIds` を人が読める形で表示する
- `videoIds` の解析に失敗した場合は、エラー状態を表示する
- プレビュー内の動画 ID は、対応するニコニコ動画 watch URL へのリンクとして表示してよい

このページは共有 URL の内容確認を目的とし、プレイリストのインポートや再生そのものはブラウザ拡張機能側で行う。

## 7. 再生仕様

### 7.1 基本再生

- プレイリストの先頭動画から再生を開始する
- 現在動画の再生終了を検知したら、次の動画ページへ遷移する
- 最後の動画まで到達したら連続再生を終了する
- プレイリスト全体のリピートが有効な場合、最後の動画まで到達したら先頭動画へ戻る
- 各動画のリピートが有効な場合、設定条件を満たすまで同じ動画を繰り返してから次の動画へ進む
- プレイリスト全体のリピートが無効な場合、最後の動画終了時に完了時オプションを実行できる
  - 音を再生する
  - 再生タブを前面に出す
  - 確認ダイアログを表示する

### 7.2 手動操作

ユーザーは以下を操作できる。

- プレイリスト内の任意の動画へ移動する
- ポップアップから別のプレイリストへ切り替える
- プレイリスト全体のリピートと各動画リピートを切り替える

### 7.3 状態復元

プレイリストと再生位置は local storage から復元する。

1 つのプレイリストは、同時に 1 つのタブにだけ紐づける。

watch ページを開いた場合、ポップアップはまずそのタブの再生コンテキストを表示する。

そのタブに再生コンテキストが存在しない場合は、最後に操作が行われたプレイリストを表示する。

タブを閉じた場合でも、対応するプレイリストの再生位置情報は storage 上に残ってよい。

popup は、生存中の再生タブが無い場合でも、保存済みの再生位置情報をもとに現在位置表示を継続してよい。

プレイリスト共有 URL は再生状態を直接変更せず、インポート確認後に保存済みプレイリストとして扱う。

## 8. ニコニコ動画ページとの連携

### 8.1 対象ページ

対象は以下とする。

```txt
https://www.nicovideo.jp/watch/*
```

### 8.2 動画 ID

ニコニコ動画の videoId を再生単位として扱う。

例:

```txt
sm9
sm12345678
so34567890
nm12345678
```

### 8.3 ページ遷移

次動画へ進む際は、以下の形式の URL へ遷移する。

```txt
https://www.nicovideo.jp/watch/{videoId}
```

プレイリスト状態は local storage で維持し、次動画への watch URL にはプレイリスト情報を引き継がない。

再生位置の引き継ぎを避けるため、次動画への遷移 URL には `?from=0` を付与する。

## 9. NiconiCompare との連携

NiconiPlaylist は NiconiCompare の評価データを直接読み取ることを必須としない。

NiconiCompare は現在、CSV/TSV/JSON などの出力を提供している。NiconiPlaylist は、これらの出力を利用してプレイリスト共有 URL を生成・インポートする。

想定される連携フローは以下。

1. NiconiCompare で動画を評価する
2. NiconiCompare から CSV/TSV/JSON を出力する
3. 外部ツール上でデータを動画 ID 列へ変換する
4. Rating / RD / 評価数などに基づいて動画 ID の順序を生成する
5. NiconiPlaylist 用のプレイリスト共有 URL を出力する
6. 拡張機能が共有 URL をインポートし、local storage へ保存してアクティブなプレイリストへ設定する

## 10. UI 仕様

NiconiPlaylist は、watch ページ上に独自 UI を常時表示しない。

UI はブラウザ拡張のポップアップとオプションページで提供する。

content script は UI 表示を担当せず、現在位置の判定、再生終了検知、次動画への遷移を担当する。

### 10.1 ポップアップ

- 現在アクティブなタブに紐づく再生コンテキスト、または最後に操作が行われたプレイリストを表示する
- プレイリスト選択を行う
  - 候補は新しく追加したプレイリストほど上に表示する
  - 保存済みの再生状態データを持つプレイリストには `▶` を付ける
  - popup 非表示に設定されたプレイリストは、現在アクティブまたは再生中でない限り候補に表示しない
- 現在位置: `3 / 20` を表示し、その位置へスクロールできる
- プレイリスト内動画一覧を表示する
- 動画をクリックしてプレイリスト内の再生位置を変更する
- プレイリスト内動画や前後移動ボタンを右クリックして、現在動画の再生終了後に移動する先を予約できる
  - 予約対象は水色でハイライト表示する
  - 同じ予約対象を再度右クリックした場合は予約を解除する
- 前の動画 / 次の動画へ移動できる
- 生存中の再生タブがあれば、丸ボタンからそのタブへフォーカスできる
- プレイリストメモを表示・編集できる
- プレイリスト全体のリピートを ON / OFF できる
- 各動画のリピート設定を切り替えできる
- 再生中のタブが存在しない時は、グローバル再生設定の `再生タブが無い時の開き方` に従って以下のいずれかの挙動をする
  - 新しいタブで開く
  - 現在のタブを上書きして開く

ポップアップでプレイリストを選択した場合、そのプレイリストに紐づくタブがあればそのタブへフォーカスする。

紐づくタブがなければ、グローバル再生設定に従って watch ページを開き、そのプレイリストの再生コンテキストを割り当てる。

ポップアップからオプションページを開いた場合、オプションページ表示後にポップアップを閉じる。

### 10.2 オプションページ

- 既定では `プレイリスト` タブを開く
- 保存済みプレイリストを管理する
  - 一覧は `createdAt` の新しいプレイリストほど上に表示する
- プレイリスト一覧から 1 件を選び、詳細を表示する
- プレイリストの title / memo / 動画一覧を編集する
- プレイリストをアクティブに設定できる
- プレイリストを先頭または現在の再生位置から再生開始 / 再開できる
- プレイリストごとの再生状態表示を手動で削除できる
- プレイリストごとに popup の候補へ表示するかどうかを切り替えできる
- プレイリストごとの作成日時、更新日時、最終再生日時、プレイリスト再生完了日時を内部保持する
- 動画を追加する
- 動画を削除予定としてマークし、保存時にまとめて反映する
- 動画順を上下移動またはドラッグアンドドロップで編集する
- 現在の再生位置を一覧内で表示する
- `動画` タブで、保存済みプレイリストに含まれる動画を playlist 横断で一覧表示する
  - 1 行 = `watchId`
  - メタデータが取得済みなら、サムネイル、タイトル、投稿日時、再生時間、投稿者名を表示する
  - メタデータ未取得の動画も、`watchId` ベースで一覧表示する
  - `タイトル` / `watchId` / `投稿者` / `playlist` で絞り込みできる
  - `タイトル` / `投稿日時` / `再生時間` / `watchId` / `投稿者` / `playlist件数` でソートできる
  - 表示中に新しいメタデータを取得しても、一覧のソート、絞り込み、スクロール位置を維持する
  - 新しいメタデータは更新通知から明示的に反映するか、動画タブを開き直した時に反映する
  - playlist 所属は一括または行ごとに展開 / 折りたたみできる
  - playlist 名をクリックすると、その playlist の詳細を `プレイリスト` タブで開ける
- 既存プレイリストの複製プレイリストを 1 操作で作成する
- 既存プレイリストから、シャッフル済みの複製プレイリストを 1 操作で作成する
- 共有 URL を発行する
- プレイリスト単位の JSON をインポート / エクスポートする
  - playlist
  - 参照される動画メタデータ
  - 参照される投稿者データ
    をセットで扱う
- 再開方法、各動画リピート用プリセット、プレイリスト完了後の動作を編集する
  - 変更は自動保存される
- データの JSON バックアップと復元を行う
- 孤立データ削除を行う
- 指定日数より前のプレイリスト再生状態データを手動で削除できる
- 全データ削除を行う

## 11. データ保持方針

プレイリストは、拡張機能の local storage に保存する。

保存対象は以下とする。

- 保存済みプレイリスト
- プレイリストごとのタブ再生コンテキスト
- プレイリスト再生関連のデバッグイベント履歴
- 最後に操作が行われたプレイリスト
- グローバルな再生設定
  - 再生タブが無い時の開き方
  - プレイリスト全体のリピート有効 / 無効
  - 各動画リピート用のプリセット一覧
  - 現在選択中の各動画リピート設定
  - プレイリスト完了後の動作
    - 通知方法
    - 通知音の音量
    - 通知音の連続再生回数
    - 再生タブ前面化の有無

プレイリスト単位の JSON エクスポートは、1 件のプレイリストに加えて、そのプレイリストが参照する動画メタデータと投稿者データを含める。

プレイリスト単位の JSON インポートは、新しい保存済みプレイリストとして追加し、含まれる動画メタデータ / 投稿者データを local storage へ取り込む。

全体の JSON バックアップには、プレイリスト再生関連のデバッグイベント履歴を含めてよい。

全体の JSON インポート時、プレイリスト再生関連のデバッグイベント履歴は復元せず、空配列で初期化してよい。

保存済みプレイリストには、`videoIds` に加えて `title` や `memo` のような省略可能な付加情報を含めてよい。

保存済みプレイリストには、popup 候補へ表示するかどうかを示す状態を保持してよい。

保存済みプレイリストには、以下の時刻情報を保持してよい。

- `createdAt`
- `updatedAt`
- `lastPlayedAt`
- `lastCompletedAt`

## 12. エラー処理

以下の場合はユーザーに分かる形で通知する。

- URL からプレイリストを解析できない
- プレイリストが空
- videoId が不正
- プレイリスト共有 URL が利用側の長さ上限を超えている
- 現在動画がプレイリスト内に存在しない
- 次動画への遷移に失敗した

不正な videoId を含む場合は、プレイリスト全体を無効とする。

## 13. セキュリティ・プライバシー

- プレイリスト共有 URL に含まれる動画 ID は共有可能な情報として扱う
- 外部サーバーへ自動送信しない
- プレイリスト共有 URL に含める情報は、動画 ID 列、タイトル、メモなど共有に必要なプレイリスト情報に限定する

## 14. 責務分担

### 14.1 拡張機能

拡張機能は、保存済みプレイリストの管理、タブごとの再生コンテキストの管理、最後に操作が行われたプレイリストの管理、ニコニコ動画 watch ページでの連続再生、再生位置の保持を担当する。

### 14.2 境界

プレイリストの正本は拡張機能の local storage に置く。
